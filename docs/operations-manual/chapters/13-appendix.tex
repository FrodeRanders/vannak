\subsection{Ipto SQL Schema}

The integration backend loads three external SQL files from
\filepath{../ipto/shared/db/postgresql}: \filepath{schema.sql} (tables),
\filepath{procedures.sql} (stored procedures), and \filepath{boot.sql} (boot
data). \vannak{} treats these as the authoritative \ipto{} schema and does not
redefine them. The relevant facts for \vannak{} are that boot data seeds tenant
id 1 (\texttt{SCRATCH}), and that provenance is stored as units carrying a
\texttt{tenantid}, a \texttt{corrid}, a status (30 = EFFECTIVE), and a list of
attributes.

\subsection{Integration Test Runbook}

\begin{enumerate}
\item Start PostgreSQL:
  \shellcmd{docker compose -f docker-compose.test.yml up -d}.
\item Run the test:
  \shellcmd{VANNAK_PG_INTEGRATION=1 cargo test --features ipto-writer --test vannak_integration -- --nocapture}.
\item Tear down:
  \shellcmd{docker compose -f docker-compose.test.yml down -v}.
\item Or run everything via
  \shellcmd{./scripts/run-integration-test.sh} (add \shellcmd{--cluster} for the
  \raft{} cluster).
\end{enumerate}

The test exercises the full ingest, index, outbox, and \ipto{} writer flow plus
a shard-range rebalance.

\subsection{Codec Reference}

\texttt{IptoWritePayload::encode} produces a small, stable, little-endian binary
format used for outbox segment records. Strings are length-prefixed
(\texttt{u32} length followed by UTF-8 bytes).

\begin{lstlisting}[numbers=none]
target:           string
shard_id:         u64
idempotency_key:  string
mapping_version:  string
attribute_count:  u32
repeated attribute_count times:
  attribute_name: string
  value:          tag(u8) + value-body
\end{lstlisting}

Value tags: \texttt{0} String (string), \texttt{1} Integer (\texttt{i64}),
\texttt{2} Boolean (\texttt{u8}), \texttt{3} Timestamp (string), \texttt{4}
StringList (\texttt{u32} count followed by that many strings). Decoding rejects
trailing bytes, unexpected end of input, invalid UTF-8, and unknown value tags.

\subsection{Segment Format}

\begin{lstlisting}[numbers=none]
magic[8]                  // "VANNAK01"
repeated:
  len: u32 little endian
  checksum: u64 little endian
  payload: [u8; len]
\end{lstlisting}

The per-record checksum is an FNV-1a derivative over the payload; the rolling
segment checksum combines records with a rotate-and-xor step.

\subsection{Glossary}

\begin{description}
\item[Process event] a \durga{} lifecycle event, ingested as a
  \texttt{PipelineEvent}.
\item[Data individual] a single flowing data item with a stable
  \texttt{DataIndividualId}.
\item[Passive metadata] facts observed at receive/create boundaries.
\item[Active metadata] facts produced by processing (transform, mask, validate,
  enrich).
\item[\texttt{DataIndividualShardId}] domain placement key selecting an \ipto{}
  instance; not a \sitas{} executor shard.
\item[Placement map] an epoch-versioned ring plus overrides, replicated through
  \raft{}.
\item[Outbox] the durable, replayable queue of pending \ipto{} writes.
\item[Segment] an append-only, checksummed durability file with the
  \texttt{VANNAK01} magic.
\item[Checkpoint] a replicated record of consumption progress for recovery.
\item[Writer lease] single-writer authority for an \ipto{} target, fenced by a
  \texttt{LeaseEpoch}.
\item[Hot path] the shard-local ingest and indexing path.
\item[Agreement path] the \raft{}-coordinated control path.
\end{description}
